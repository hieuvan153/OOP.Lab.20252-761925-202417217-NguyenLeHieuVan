Question (Section 11): The Aims class must be updated to handle any exceptions generated when the play() methods are called. What happens when you don't update for them to catch?

Answer:
Since PlayerException inherits directly from `java.lang.Exception` (making it a checked exception), the Java compiler enforces that it must be explicitly handled. If you do not update the Aims class to catch this exception using a try-catch block (or delegate it using the `throws` keyword), your code will fail to compile. The compiler will output an error stating: "unreported exception PlayerException; must be caught or declared to be thrown".

If the exception were unchecked (i.e., inheriting from RuntimeException) and you did not catch it, the program would compile but would crash at runtime whenever a media item with a non-positive length is played. This would abruptly terminate the thread and print a stack trace to the console. Catching the exception allows the GUI application to handle the error gracefully, such as by showing an alert dialog to the user as requested in the lab, instead of crashing the entire application.
